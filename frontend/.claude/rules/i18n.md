# i18n — custom hook, NOT next-intl

`next-intl` is in `package.json` but **not used**. The actual system:

- Catalogs: `src/locales/{fr,en}.json` — must stay in lockstep (the same keys in both).
- Provider: `src/context/LanguageContext.jsx` — persists choice in `localStorage` under `locale`, sets `document.documentElement.lang`.
- Hook: `src/hooks/useI18n.js` exposes `useTranslation()` → `{ t, locale }`.
- Usage: `t("history.resultsCount", { count: 5 })`. Supports `{var}` interpolation in the translation string.
- Fallback chain: missing key in current locale → fall back to `fr` → fall back to the raw key.

When adding UI strings, **add the key to both `fr.json` and `en.json`**. The language switcher lives in **Settings → Préférences** and calls `useLanguage().changeLocale(value)`.
